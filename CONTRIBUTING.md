# Contributing to ServerDoctor

Thank you for your interest in contributing to ServerDoctor.

## Reporting Bugs

Before opening a bug report:

1. Make sure you are using the latest release.
2. Search existing issues to avoid duplicates.
3. Include relevant logs, stack traces, and reproduction steps.

## Suggesting Features

Feature requests are welcome. Please:

* Explain the problem you are trying to solve.
* Describe your proposed solution.
* Consider alternative approaches.

## Development Setup

Requirements:

* Java 21
* Gradle

Clone the repository:

```bash
git clone https://github.com/Shvquu/server-doctor.git
cd server-doctor
```

Build the project:

```bash
./gradlew build
```

## Pull Requests

Before submitting a pull request:

* Ensure the project builds successfully.
* Follow the existing code style.
* Keep pull requests focused and small where possible.
* Update documentation if necessary.

## Commit Messages

Recommended format:

```text
feat: add player diagnostics
fix: resolve database connection issue
docs: update README
refactor: simplify storage implementation
```

## Code Review

All pull requests may be reviewed before merging. Feedback should be addressed constructively and respectfully.

Thank you for helping improve ServerDoctor.
