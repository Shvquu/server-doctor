# Security Policy

## Supported Versions

The latest release is considered supported and receives security updates when necessary.

| Version        | Supported |
| -------------- | --------- |
| Latest Release | ✅         |
| Older Releases | ❌         |

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Instead, contact the maintainer privately and include:

* A description of the vulnerability
* Steps to reproduce
* Potential impact
* Any suggested remediation

Security reports will be reviewed as quickly as possible.

## Response Process

When a valid vulnerability is confirmed:

1. The issue will be investigated.
2. A fix will be developed and tested.
3. A security release will be published.
4. Credit may be given to the reporter if desired.

## Responsible Disclosure

Please allow reasonable time for vulnerabilities to be addressed before publicly disclosing them.
